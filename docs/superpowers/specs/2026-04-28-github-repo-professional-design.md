## Section 1 — Git & Security

### Git initialization
- Run `git init` in the project root
- Set default branch to `main`

### `.gitignore`
Exclude the following:
- `.env` — contains live API keys (CRITICAL: must never be committed)
- `venv/` — Python virtual environment
- `data/myco.db` — runtime SQLite database
- `data/skills/` — Python skills generated by agents at runtime via `exec()`
- `__pycache__/`, `*.pyc`, `*.pyo`
- `.DS_Store`, `Thumbs.db`
- `*.egg-info/`, `dist/`, `build/`

### Safe files to commit
- `.env.example` — template with placeholder values, no real keys
- `data/.gitkeep` — keeps the `data/` directory in the repo

### Initial commit
Single commit with all project files (excluding the above), message: `feat: initial commit — Myco organism operating system`

## Section 4 — GitHub Actions CI

**File:** `.github/workflows/ci.yml`  
**Triggers:** push and pull_request to `main`  
**Python version:** 3.11

**Steps:**
1. Checkout code
2. Set up Python 3.11
3. Install dependencies (`pip install -r requirements.txt`)
4. Verify app imports cleanly (`python -c "import main"`)
5. Smoke test: start server in background, wait for it to be ready, `GET /` returns 200, kill server

**Environment:** CI runs with `OPENROUTER_API_KEY=test` and `DATABASE_URL=sqlite:///./data/test.db` to avoid needing real credentials.

## Section 5 — Issue & PR Templates

### `.github/ISSUE_TEMPLATE/bug_report.md`
Fields:
- Description of the bug
- Steps to reproduce
- Expected vs actual behavior
- AI model being used (`OPENROUTER_MODEL` value)
- Python version

### `.github/ISSUE_TEMPLATE/feature_request.md`
Fields:
- Problem this feature solves
- Proposed solution
- Alternatives considered

### `.github/pull_request_template.md`
Checklist:
- Description of change
- Type (bug fix / feature / refactor / docs)
- How to test locally
- Does it require `.env` changes?
